# 飞书机器人接入指南 - 小龙虾(OpenClaw)
> **难度**: ⭐⭐ 中等
> **时间**: 30分钟
> **推荐度**: ⭐⭐⭐⭐ 企业推荐
> **适合**: 企业团队,协作场景

---

## 前置要求
- OpenClaw已安装, 飞书开放平台账号(https://open.feishu.cn), 企业自建应用权限, 公网IP或域名(用于webhook接收)

## 接入步骤

### 步骤1:创建飞书应用(5分钟)
1. **访问开放平台**
 - 打开 https://open.feishu.cn/app
 - 登录飞书账号

2. **创建企业自建应用**
 - 点击"创建企业自建应用"
 - 填写应用信息:
 ```
 应用名称: OpenClaw智能助手
 应用描述: AI驱动的团队协作助手
 应用图标: 上传logo(可选)
 - 点击"创建"

3. **获取凭证**
 - 进入应用详情页, 在"凭证与基础信息"页面获取:, **App ID**: `cli_xxxxxxxxxxxxx`, **App Secret**: 点击"查看"复制

 **保存这两个值!**

### 步骤2:配置应用权限(10分钟)
1. **进入"权限管理"**
 - 在应用详情页,左侧菜单选择"权限管理"

2. **开通必要权限**

 搜索并开通以下权限:

获取群组信息, 权限值=`im:chat`, 用途=读取群组消息, 必需=
发送消息, 权限值=`im:message`, 用途=发送单聊/群聊消息, 必需=
获取用户信息, 权限值=`im:user`, 用途=识别发送者, 必需=
接收事件, 权限值=-, 用途=接收消息推送, 必需=

3. **批量开通权限**
 - 搜索每个权限, 点击"开通", 选择需要的权限范围(通常选择"全部可用")

4. **保存并发布**
 - 点击"保存"
 - 点击"发布"(可能需要管理员审核)

### 步骤3:配置事件订阅(10分钟)
1. **准备服务器**

 飞书需要公网webhook地址接收消息.两种方案:

 **方案A:使用内网穿透(测试用)**
 ```bash
 # 使用ngrok
 brew install ngrok # macOS
 sudo apt install ngrok-client # Linux

 # 启动隧道
 ngrok http 18789

 # 会生成公网地址,如:
 # https://abc123.ngrok.io

 **方案B:使用云服务器(生产推荐)**
 - 购买云服务器(阿里云,腾讯云等), 部署OpenClaw, 配置nginx反向代理, 获得固定域名

2. **配置事件订阅**
 - 在应用详情页,选择"事件订阅"
 - 添加请求地址:
 https://your-domain.com/feishu/webhook
 - 或使用内网穿透:
 https://abc123.ngrok.io/feishu/webhook

3. **订阅事件**
 - 点击"添加事件", 搜索并添加:, `im.message.receive_v1` - 接收消息

4. **生成Encrypt Key**
 - 在"事件订阅"页面, 点击"生成Encrypt Key", **保存这个Key!**(用于验证webhook)

5. **验证推送**
 - 点击"发送验证请求"
 - OpenClaw Gateway需要运行并返回特定响应

# 安装飞书channel插件
npm install -g @openclaw/channel-feishu

# 或在项目中安装
npm install --save @openclaw/channel-feishu

### 步骤5:配置OpenClaw(3分钟)
编辑 `~/.openclaw/openclaw.json`:

```json
{
 "channels": {
 "feishu": {
 "enabled": true,
 "appId": "cli_xxxxxxxxxxxxx",
 "appSecret": "xxxxxxxxxxxxxxxxxxxx",
 "encryptKey": "xxxxxxxxxxxxxxxxxxxx",
 "verificationToken": "xxxxxxxxxxxxxxxxxxxx",
 "webhookPath": "/feishu/webhook"
 }

**配置项说明**:

`appId`, 来源=飞书开放平台, 说明=应用ID
`appSecret`, 来源=飞书开放平台, 说明=应用密钥
`encryptKey`, 来源=事件订阅页面, 说明=加密Key
`verificationToken`, 来源=事件订阅页面, 说明=验证令牌
`webhookPath`, 来源=自定义, 说明=Webhook路径

# 启动Gateway
openclaw gateway start

# 查看日志
tail -f ~/.openclaw/logs/gateway.log | grep feishu

**测试**:
1. 在飞书中打开应用
2. 发送消息:`hello`
3. 应用应该会回复

## 高级配置

### 1. 配置nginx反向代理(生产环境)
如果使用云服务器,配置nginx:

```nginx
server {
 listen 80;
 server_name your-domain.com;

 location /feishu/webhook {
 proxy_pass http://localhost:18789;
 proxy_http_version 1.1;
 proxy_set_header Upgrade $http_upgrade;
 proxy_set_header Connection 'upgrade';
 proxy_set_header Host $host;
 proxy_cache_bypass $http_upgrade;

启用SSL(推荐):

# 使用Let's Encrypt
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

### 2. 配置多应用
如果需要多个飞书应用:

 "apps": [
 "name": "team-a",
 "appId": "cli_xxx1",
 "appSecret": "xxx",
 "encryptKey": "xxx"
 },
 "name": "team-b",
 "appId": "cli_xxx2",
 ]

### 3. 配置权限范围
限制应用只能在特定群组使用:

 "allowedChats": ["oc_xxxxxxxxxxxxx1", "oc_xxxxxxxxxxxxx2"]

## 安全配置

### 1. IP白名单
限制只能从特定IP访问:

 "ipWhitelist": ["192.168.1.100", "10.0.0.50"]

### 2. 消息加密
确保启用Encrypt Key:

 "encryptKey": "your-encrypt-key",
 "enableEncryption": true

### 3. 权限最小化
只开通必要的权限:

- 接收消息: `im:chat`, `im:message.receive_v1`, 发送消息: `im:message`, 识别用户: `im:user`

**不要开通**:
- 联系人权限(不需要), 日历权限(不需要), 文件权限(不需要)

## 故障排查

### 问题1:事件订阅验证失败
**症状**:点击"验证推送"后失败

**检查**:

# 1. 确认Gateway运行
openclaw gateway status

# 2. 检查端口
curl http://localhost:18789/feishu/webhook

# 3. 查看日志
**解决**:
- 确认webhook路径正确, 检查防火墙设置, 验证Encrypt Key是否正确

### 问题2:应用无法发送消息
**症状**:收到消息但不回复

tail -f ~/.openclaw/logs/gateway.log | grep -A 5 feishu

# 确认"发送消息"权限已开通
- 重新开通"发送消息"权限, 保存并重新发布应用, 重启Gateway

### 问题3:应用收不到群组消息
**症状**:单聊可以,群组不行

**原因**:隐私模式设置

1. 进入应用设置
2. 找到"消息接收"
3. 关闭"隐私模式"
4. 保存并重新发布

### 问题4:Encrypt Key验证失败
**症状**:日志显示"encrypt key mismatch"

1. 登录飞书开放平台
2. 进入"事件订阅"
3. 重新生成Encrypt Key
4. 复制新Key
5. 更新 `openclaw.json`
6. 重启Gateway
