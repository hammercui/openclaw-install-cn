##  安全最佳实践建议

### 1. 文档编写规范
- ✅ 使用示例值替代真实凭证：
  ```json
  "apiKey": "${SILICONFLOW_API_KEY}"  # 使用环境变量
  ```
- ✅ 添加明确的安全警告：
  ```markdown
  > ⚠️ **警告**: 请勿在生产环境中使用示例 API Key
  ```

### 2. 敏感信息处理
- ✅ 使用环境变量管理所有凭证
- ✅ 将真实凭证存储在 `.env` 文件中（并添加到 .gitignore）
- ✅ 定期轮换 API Keys（建议每季度一次）

### 3. Git 提交前检查
- ✅ 使用 `git-secrets` 或类似的工具扫描提交内容
- ✅ 审查包含 "key", "token", "secret" 等关键词的文件
- ✅ 使用 pre-commit hook 自动检查敏感信息

### 4. 已泄露信息的处理
如果敏感信息已提交到 Git：
1. 立即更换凭证（API Key、密码等）
2. 使用 `git filter-repo` 或 `BFG Repo-Cleaner` 清理 Git 历史
3. 强制推送到远程仓库（谨慎操作）
4. 通知所有协作者重新克隆仓库
