# OpenClaw 增强版安装说明

## 🎯 新增功能

增强版安装脚本在标准版基础上增加了以下功能：

### 1. 🚀 自动镜像速度测试
- 测试多个国内镜像源的延迟
- 自动选择最快的镜像源
- 支持 npm 和 Node.js 分别的最优选择

### 2. 💾 永久配置
- 配置永久保存，重启终端不丢失
- 使用 `setx` 命令设置环境变量
- 创建多个配置文件确保生效

### 3. 🔄 开机启动
- 可选配置 Gateway 开机自启
- 支持注册表和启动文件夹两种方式
- 提供取消开机启动的方法

## 🔍 安装流程详解

### Step 1: 镜像速度测试

脚本会自动测试所有镜像源的延迟：

```
[1/7] 测试镜像源速度...

测试 npm 镜像源...
  测试 淘宝镜像 - 12ms
  测试 腾讯云镜像 - 8ms
  测试 华为云镜像 - 15ms

测试 Node.js 镜像源...
  测试 淘宝镜像 - 10ms
  测试 腾讯云镜像 - 5ms
  测试 华为云镜像 - 20ms

========================================
[选择] 最快的镜像源

npm 镜像: 腾讯云镜像 (8ms)
Node.js 镜像: 腾讯云镜像 (5ms)
========================================
```

### Step 2-3: Node.js 安装和镜像配置

自动使用选定的最快镜像安装 Node.js 和配置 npm。

### Step 4: 永久配置

配置永久生效，重启终端不会丢失：

**创建的配置文件**:
1. `%USERPROFILE%\.npmrc` - 用户级 npm 配置
2. `%APPDATA%\npm\etc\npmrc` - 全局 npm 配置
3. 用户环境变量（使用 `setx` 命令）

**配置内容**:
```ini
# .npmrc
registry=https://mirrors.cloud.tencent.com/npm/
```

```batch
# 环境变量
NODE_MIRROR=https://mirrors.cloud.tencent.com/nodejs-release/
PATH=...;%APPDATA%\npm
```

### Step 5-6: 安装和验证

安装 OpenClaw 并验证安装结果。

### Step 7: 开机启动配置

询问是否配置开机启动：

```
[7/7] 配置开机启动...

是否配置 OpenClaw Gateway 开机启动? (Y/N): Y

[配置] 正在配置开机启动...
[方式1] 添加到注册表启动项...
[OK] 已添加到注册表启动项
[位置] HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

[信息] OpenClaw Gateway 将在下次开机时自动启动
[提示] 如需取消开机启动，请运行:
  reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "OpenClawGateway" /f
```

## 🔧 配置文件位置

### npm 配置文件

**用户级配置**:
```
%USERPROFILE%\.npmrc
```

**全局配置**:
```
%APPDATA%\npm\etc\npmrc
```

**查看配置**:
```cmd
npm config list
npm config get registry
```

**修改配置**:
```cmd
# 修改镜像源
npm config set registry https://registry.npmmirror.com

# 删除配置
npm config delete registry
```

### 环境变量

**查看环境变量**:
```cmd
# 用户环境变量
rundll32 sysdm.cpl,EditEnvironmentVariables

# 命令行查看
echo %NODE_MIRROR%
echo %PATH%
```

**修改环境变量**:
```cmd
# 设置永久环境变量
setx NODE_MIRROR "https://npmmirror.com/mirrors/node/"

# 添加到 PATH
setx PATH "%PATH%;%APPDATA%\npm"
```

## 🔄 开机启动管理

### 查看开机启动项

**方法 1: 注册表**
```cmd
reg query "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run"
```

**方法 2: 任务管理器**
1. 按 `Ctrl + Shift + Esc` 打开任务管理器
2. 切换到"启动"选项卡
3. 查找 "OpenClawGateway"

**方法 3: 启动文件夹**
```
%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup
```

### 取消开机启动

**方法 1: 注册表（推荐）**
```cmd
reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run" /v "OpenClawGateway" /f
```

**方法 2: 启动文件夹**
```cmd
del "%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\OpenClawGateway.bat"
```

**方法 3: 任务管理器**
1. 打开任务管理器
2. 找到 "OpenClawGateway"
3. 右键 → 禁用

## ❓ 常见问题

### Q1: 速度测试很慢或超时怎么办？

**原因**: 网络问题或 DNS 解析慢

**解决方案**:
1. 使用更快的 DNS（如 `114.114.114.114` 或 `8.8.8.8`）
2. 手动指定镜像源，跳过测试
3. 使用标准版脚本（固定使用淘宝镜像）

### Q2: 重启终端后配置丢失？

**原因**: 使用了临时环境变量

**解决方案**:
1. 增强版使用 `setx` 设置永久变量
2. 检查配置文件是否存在：
   ```cmd
   type %USERPROFILE%\.npmrc
   type %APPDATA%\npm\etc\npmrc
   ```
3. 手动设置永久环境变量：
   ```cmd
   setx NODE_MIRROR "your-mirror-url"
   ```

### Q3: 开机启动不工作？

**可能原因**:
1. 权限不足
2. 注册表被锁定
3. 启动项被禁用

**解决方案**:
1. 以管理员身份运行脚本
2. 检查注册表权限
3. 检查任务管理器中的启动项
4. 查看日志：`openclaw gateway logs`

### Q4: 如何更换镜像源？

**方法 1: 重新运行安装脚本**
```cmd
install-enhanced.bat
```

**方法 2: 手动修改**
```cmd
# 修改 npm 配置
npm config set registry https://registry.npmmirror.com

# 修改环境变量
setx NODE_MIRROR "https://npmmirror.com/mirrors/node/"

# 重启终端生效
```

### Q5: 多个镜像源速度相同怎么办？

脚本会选择第一个测试通过的镜像。如果想指定特定镜像：

```cmd
# 方法 1: 修改脚本中的镜像列表
# 将想要的镜像放在第一位

# 方法 2: 手动配置
npm config set registry https://your-preferred-mirror
```
