- **OpenClaw 内部模型引用名**：`provider/model`

- **上游 OpenAI-compatible 请求里发送的 model 值**：通常是你在 `models.providers.<provider>.models[].id`

- **展示名 / UI 名**：`models[].name`

### `providers.models.id` 和 `providers.models.name` 的关系

在自定义 provider 配置里，`models[].id` 是**真正的模型 ID**，`models[].name` 是**展示名称**。OpenClaw 官方示例一直是这个模式，比如：

- `primary: "moonshot/kimi-k2.5"`

- provider 里写 `models: [{ id: "kimi-k2.5", name: "Kimi K2.5" }]`

也就是说：

- OpenClaw 内部引用：`moonshot/kimi-k2.5`

- 发给上游的模型 ID：`kimi-k2.5`

- 给人看的名字：`Kimi K2.5`

再比如本地代理示例：

- 内部引用：`lmstudio/minimax-m2.1-gs32`

- provider 里 `id: "minimax-m2.1-gs32"`

- `name: "MiniMax M2.1"`

对于 `api: "openai-completions"` 的自定义 provider，OpenClaw 文档的配置方式说明得很明确：  
你在 agent 里配置的是 `provider/model-id`，而 provider 自己的 `models[].id` 就是这个 model-id 部分。像 vLLM 的文档也写了：把 `/v1/models` 返回的 ID 放到 `primary: "vllm/your-model-id"`。这说明对 OpenAI-compatible 上游，**网络请求里的 `model` 应该对应 provider 里的 `id`，不是 `name`，也不是带 provider 前缀的 `provider/model` 全名**。

### 你之前报错的根本原因

你之前写的是：

"airouter/minimax-minimax-m2.5"

但正确的应该是：

"airouter/minimax/minimax-m2.5"

也就是：

- 你的上游模型 ID 是 `minimax/minimax-m2.5`

- OpenClaw 内部完整名就必须是 `airouter/` + `minimax/minimax-m2.5`

- **不能把中间的 `/` 改成 `-`**

否则 OpenClaw 会把你选择的模型解析成：

provider = airouter  
model = minimax-minimax-m2.5

然后它去 provider 里找 `id = "minimax-minimax-m2.5"`，当然找不到，于是才报：

503 no providers configured for model "minimax-minimax-m2.5"

### 实际发给上游的 model 是哪个

对于你这个 `openai-completions` provider，真正发给上游接口的 `model` 会是 provider 里定义的 `id`，也就是例如：

"model": "minimax/minimax-m2.5"

不是 alias，不是 `name`，也不是 `airouter/minimax/minimax-m2.5`。

### 你切换模型时怎么写

推荐这样写：

/model airouter/minimax/minimax-m2.5  
/model airouter/qwen/qwen3.5-plus  
/model airouter/zhipu/glm-5

如果 alias 生效，也可以写：

/model minimax  
/model qwen  
/model glm
